Question 20

Your on-premises network has an IP address range of 10.0.0.0/23.

You have an Azure virtual network named VNet1 that contains a virtual machine named VM1.

VNet1 has an IP address range of 10.0.1.0/24.

You need to deploy a Site-to-Site (S2S) VPN to connect the on-premises network to VNet1.

What should you do first?

Explanation:

There is an overlap in the ranges 10.0.0.0/23 and 10.0.1.0/24 as the 10.0.1.0/24 range is entirely contained within the 10.0.0.0/23 range. Configure VNet1 to use the IP address range of 10.0.2.0/24. This step ensures that there are no overlapping IP ranges between the on-premises network and the Azure virtual network, which is essential for a successful S2S VPN connection.
